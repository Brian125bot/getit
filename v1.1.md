# Specification: `getit` v1.1 (Stabilization & Distribution)

**Status:** Active Execution Plan for Final Polish

**Goal:** Transition the codebase from a feature-complete Release Candidate (RC) to a globally distributable, zero-leak terminal utility.

## Part 1: Codebase Pruning & Tech Debt Eradication

*The file tree reveals leftover artifacts from Phase 1. These must be purged to guarantee the security matrix.*

### 1.1 Complete Purge of Legacy Banned Paths

* **Current State:** `src/security/banned-paths.ts` is obsolete and mostly replaced by `path-policy.ts`.
* **Action Required:** 
  1. Hard-delete `src/security/banned-paths.ts`.
  2. Audit `src/tools/` to ensure 100% of execution requests route exclusively through `assertPathAllowed()` from `src/security/path-policy.ts`.
  3. Update or delete legacy tests (`tests/safety.test.ts`) that rely on the deleted module, ensuring they now test the `path-policy.ts` facade.

### 1.2 Guarantee Pure Async File Primitives

* **Current State:** File manipulations in legacy tool files often rely on `fs.writeFileSync` to avoid race conditions.
* **Action Required:** Enforce the usage of `node:fs/promises` across the entire `src/tools/` and `src/workspace/` directories. Utilize atomic UUID temp-file swapping (`rename` over temporary writes) to guarantee that async event loops are never blocked by heavy read/writes.

### 1.3 Secure the Token Stream Interceptor

* **Current State:** `src/agent/loop.ts` streams raw LLM tokens directly to `process.stdout`.
* **Action Required:** Verify that the real-time token stream is piped through `src/security/scrubber.ts`'s stateful masking session *before* hitting the terminal output, preventing high-entropy keys from leaking during conversational markdown generation.

## Part 2: Terminal UX Refinement

*With the backend fully stateful, the UI must match the fluidity of modern CLI tools.*

### 2.1 The Pre-Filled "Edit" (`e`) Mode

* **Requirement:** When a user selects `e` (Edit) on the MITL (Man-In-The-Loop) confirmation card, they should not have to retype a 5-line bash payload from scratch.
* **Action Required:** Modify `src/mitl/interceptor.ts`. When `e` is captured, use `readline` stream manipulation (`rl.write(existingPayload)`) to pre-populate the terminal prompt with the LLM's generated payload. This allows the user to simply use their arrow keys to modify a single flag before pressing Enter.

### 2.2 Mid-Turn "Clarify" (`c`) Protocol

* **Requirement:** Add a fourth option to the MITL gate: `[Y/n/e/c]`.
* **Action Required:** If the user selects `c` (Clarify), pause the tool loop and prompt: `Question for Agent ❯`. Capture the user's text, append it to the chat history as a user-role message (e.g., *"User paused execution to ask: <text>"*), and trigger a new completion generation without dropping the current execution context.

## Part 3: Distribution & Publishing Pipeline

*The project is strictly zero-dependency, making it perfect for global NPM execution via `npx`.*

### 3.1 `npx` / NPM Registry Initialization

* **Requirement:** Users should be able to run the tool instantly on any Node 20+ environment using `npx getit <prompt>`.
* **Action Required in `package.json`:**
  * Ensure `"bin": { "getit": "./dist/src/index.js" }` is correctly mapped.
  * Ensure `"type": "module"` is declared (if utilizing ESM).
  * Verify the `files` array or `.npmignore` correctly excludes `/tests` and raw `.ts` files, packaging only `/dist`, `README.md`, and `LICENSE`.

### 3.2 Executable Shebang & Argument Parsing

* **Action Required:** 
  * Inject `#!/usr/bin/env node` at the very top of `src/index.ts`.
  * Ensure global argument flags (e.g., `getit --dry-run "install ripgrep"`) are robustly parsed on initialization so the agent knows whether to enter the REPL or execute a one-shot task.

## Part 4: Deferred Features (Post-v1.1 / v2.0 Roadmap)

*As noted in `finalphase.md`, these items are strictly blocked from the current v1.1 stabilization sprint to prevent scope creep, but form the architectural blueprint for v2.0.*

1. **Native `gh` CLI Auto-Syncing:** Utilizing the local GitHub CLI (`gh`) to automatically configure OAuth tokens and push the `shadow-store.ts` backups to a private GitHub repository without requiring manual `git push` commands.
2. **Deep Profile Merging:** Enhancing `src/workspace/profiles.ts` to support deep JSON-merge precedence (e.g., configurations in `common/` are automatically overridden by identical keys in `profiles/desktop-x86_64/`).
3. **Visual Progress Spinners on Spawn:** Integrating `src/ui/spinner.ts` into `src/execution/async-process.ts` so long-running processes (like compiling from source) display an active loading state over the streaming text.

## 🛠️ Definition of Done (v1.1 Release Ready)

1. `npm run build` cleanly outputs the `/dist` directory with zero TypeScript compilation errors.
2. `npm run test` executes 100% of the suites (Carriers, Workspace, Drift, Safety, Execution) with zero failures.
3. `grep -rn "banned-paths" src/` returns absolutely zero imports or usages.
4. Installing the module locally via `npm link` allows the global `getit` command to execute successfully in the terminal.
